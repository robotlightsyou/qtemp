scope and functions

Here is the script we will be modifying in the next cahllenge, but first I want to walkthrough some new components and concepts.

The first thing we see is the declaration of a global variable. Global is a reference to the scope, and it means that the variable can be referenced from anywhere in the script. I mentioned before that a scope roughly is the level of indentation, but that's not entirely true. When we use certain keywords we create a new layer in our execution for an action to occur. In this script we see this carried out with the repeat command. Everything that exists between repeat and end repeat represents that scope. Variables that are declared within that block are only available within that block. Most of our scripts will be short enough that this won't be a large concern, but it is something we must be aware of.

if/else statements also create scope. You start with an if conditional. If that ccondition is true the enclosed block of code will be executed on your selected cue. If not you can proceed to an additional condition using "else if". Last you have the option of including an else statement. If your conditions aren't met you can execute a block of code with the else statement, or you can leave the selected cue alone by omitting the else statement.

Also introduced in this script are subroutines, or functions in other programming languages. A function is a block of code you want to call multiple times. If we were only getting input from the user once we would't need to separate this into a subroutine. But, since this functionality is highly reusable in other scripts we write it into a funtion for modularity so we can reuse it elsewhere.
