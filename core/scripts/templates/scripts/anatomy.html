anatomy of an applescript

First is the tell block. The tell block tells applescript what program to send commands to.

For some applications the tell block will only consist of the application. As we can see, this tell block also mentions the front window specifically. While you can target any open workspace, most of the scripts we'll cover her focus on the front workspace. For the sake of simplicity we'll consider the full tell block as pictured to be the standard tell block for QLab 4. You will also notice at the end of the tell block is a line that says end tell. End lines close off a level of scope. We'll talk more about scope when we discuss variables, the takeaway is that every level of indentation requires and end line.

The next line we see is a comment. Single line comments begin with --. This is simply a note for the author to themselves, or any other future reader. They're normally used to explain how a section of code works. You can put whatever you want in a comment as they are ignored by the compiler. You can also create a multiline comment using
(*this is a
multiline comment*)

Terms with blue syntax highlighting are part of standard suite. These are actions or classes that should be available across all applications that support applescript.

Terms with purple syntax highlighting are part of application suite. Keep in mind different applications may reuse the same terms, however it's possible they will be implemented in a different manner. This is a minor worry for me since I mostly script for QLab.

Terms with green syntax highlighting are part of variables. As you can see in the comment, the syntax for declaring a variable is set variable_name to value. Variables are available to the scope in which they are declared, and any scopes that are inside it. I'll unpack that a little more when we have a more complex example.
