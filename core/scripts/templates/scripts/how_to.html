how to edit applescripts

the script editor

This site focuses on adapting existing code into new tools to suit individual needs. On this page we'll walk through some of the common resources from writing scripts. The primary tools are the script editor and the library. Let's take a look at those.

Here we see an empty script editor(in dark mmode). At the top are seveal buttons. I encourage you to explore them fully, however we will only be using Play, Compile, and potentially Stop.

At the bottom of the editor there are three icons on the left side. The third option is to show or hide log.

When we show the log we get some additional options in the middle of the editor. The only one I ever use is Events, and I always use it. This gives a detailed log of every action taken in the execution of a script, handy for debugging.

Now you're ready to start typing. The large box on the top half of the window is for inputting your script. Once you have completed you can press the hammer button to compile, or use the CMD+K keyboard shortcut. If you don't receive any errors you're ready run the script using the play button, or the CMD+R keyboard shortcut.

the library

Included with the script editor is the library. Navigate to the Windows menu of the editor and select Library from the dropdown to view it. Or use the keyboard shortcut SHIFT+CMD+L

When you open the library for an application you'll see two sections, a navigation browser at the top and a list of definitions on the bottom. In the navigation browser on the left you'll find an entry for the Standard suite and one for QLab(assuming that is the library file you opened). Objects and actions in the standard suite should work with most software on a mac. I say should and most because not all developers provide thorough applescript support, but technically the standard commands should provide basic functionality. What's more interesting for us is the QLab suite.

The QLab suite provides a robust sset of objects and commands to manipulate your showfile. Associated with each object are a set of properties you can modify. Let's look at the entry for cue.

Cue's have a lot of properties. It's good to familiarize yourself with all of them, but there's no need to memorize everything, you can always look the specific name or syntax up. The 7 options that start with the lettr q are usual suspects, but several of the otheres I've never used. Next to a parameter you'll see a datatype in parenthesis. This list is not exhaustive, but some types to be aware of are:

Datatype Definition
Integer A whole number without decimals or fractions, such as 1, or 9.
Real A number with a decimal, such as 1.0 or 9.9
Text Written characters. Important for distinguishing "1" from 1.
Boolean True or False

Datatypes are important to pay attention to. Often times an error is simply because you are trying to manipulate a text when you think it is an integer. I hope as you scroll through the QLab suite you see what a powerful tool this can be with so many options at our control.

You've had a lot thrown at you fast. As we move into the exercises we'll slow down and examine the constituent pieces more closely.

With a basic understanding of what you're looking at, we're now ready to start hacking with a script. As we go through our exercises we'll cover the component parts, how to modify, and even how to write simple scripts from scratch.
